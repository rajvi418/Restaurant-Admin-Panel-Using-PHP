# Restaurant-Admin-Panel-Using-PHP
A restaurant admin panel built with PHP on a XAMPP server manages menu, orders, and users from one dashboard. It uses MySQL for data storage and supports real-time order tracking and updates. Designed with HTML/CSS, it runs locally for easy testing and efficient restaurant operations.

---

📌 Project Overview

This project is a Restaurant Admin Panel built using PHP and MySQL, running on a local XAMPP server. It provides a centralized system where administrators can control daily restaurant operations such as managing menu items, tracking customer orders, and handling basic user data.

---

⚙️ Tech Stack
Backend: PHP
Database: MySQL
Server: Apache (XAMPP)
Frontend: HTML, CSS, JavaScript

---

🚀 Features
Add / Edit / Delete Menu Items
Order Management (status updates)
Admin Login System
User Management
Simple Dashboard

---

The system focuses on simplifying backend tasks. Instead of manually updating menus or tracking orders offline, the admin can perform all actions through a web-based dashboard. Menu items can be added, edited, or removed dynamically, while orders are displayed in real time with status updates like pending, preparing, and completed.

The application uses PHP for server-side logic and MySQL for structured data storage. It connects the interface with the database to ensure smooth data handling and quick updates. The frontend is designed using HTML and CSS to keep the interface simple and easy to use.

Running on XAMPP allows the project to be tested locally without requiring a live hosting environment. This makes it suitable for learning, development, and small-scale implementations.

Overall, the project demonstrates how a basic admin panel can streamline restaurant operations by organizing data, reducing manual work, and improving efficiency through a structured system.
